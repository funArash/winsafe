Expand description

This trait is enabled with the user feature, and provides methods for HWND.

Prefer importing this trait through the prelude:

use winsafe::prelude::*;

fn MessageBox(self, text: &str, caption: &str, flags: MB) -> WinResult<DLGID>

MessageBox method.

Consider using the more modern TaskDialog method.

Examples

A modal message box, which blocks its parent:

use winsafe::prelude::*;
use winsafe::{co, HWND};

let my_hwnd: HWND; // initialized somewhere

hwnd.MessageBox("Hello, world", "title",
    co::MB::OKCANCEL | co::MB::ICONINFORMATION)?;

Usually the message box has a valid parent window, however, if for some reason you don’t have a window to serve as parent, you still can show a non-modal, parent-less message box by retrieving the desktop handle:

use winsafe::prelude::*;
use winsafe::{co, HWND};

HWND::GetDesktopWindow()
    .MessageBox("Hello, world", "Title", co::MB::ICONEXCLAMATION)?;

fn SendMessage<M>(self, msg: M) -> M::RetType where
    M: MsgSend, 

SendMessage method.

The return type is variable, being defined by the RetType associated type of the MsgSend trait. That means each message can define its own return type.

Examples

Sending a bm::GetImage button message, which demands an image type parameter. Note that this specific message can also return an error, which is handled with ?:

use winsafe::prelude::*;
use winsafe::{co, HWND, msg::bm};

let hwnd: HWND; // initialized somewhere

let bmp = hwnd.SendMessage(bm::GetImage {
    img_type: co::IMAGE_TYPE::BITMAP,
})?;

Sending an em::CharFromPos edit message, which receives point coordinates and returns two values:

use winsafe::prelude::*;
use winsafe::{HWND, msg::em, POINT};

let hwnd: HWND; // initialized somewhere

let (char_pos, line_pos) = hwnd.SendMessage(
    em::CharFromPos {
        coords: POINT::new(12, 20),
    },
);
